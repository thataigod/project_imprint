# Antigravity Agent Instructions

These instructions were provided by the USER to govern Antigravity's behavior for this repository.

## CI/CD and Commit Workflow Constraints

Before taking any actions to commit, push, or merge code, Antigravity MUST adhere to the following checklist:

### Pre-Commit Checklist (To `develop` or feature branches)
1. **Local Testing:** Ensure that `pytest` is run locally and all tests pass (using the `-s` flag on Windows to prevent stdout/stderr capture conflicts).
2. **Local Linting & Typing:** Ensure that `ruff` and `mypy` run locally and pass without errors.
3. **Local Security Audit:** Ensure that `pip-audit -r requirements.lock` is run locally and reports no vulnerabilities.
4. **Pre-commit Hooks:** Ensure `pre-commit` hooks are configured and run successfully.

### Pre-Push Checklist (To `develop` or feature branches)
1. **Verification:** Confirm that the local testing, linting, typing, auditing, and pre-commit hook runs described above were successful.
2. **Coverage:** Production files must maintain 100% test coverage (`--cov-fail-under=100`), if it falls below 100% it should be fixed before committing.
3. **Push:** Push the commit to the remote repository.

### Pre-Merge Checklist (To `master` branch)
1. **Online Verification:** Use the `gh` CLI (e.g., `gh run list`) to verify that the online CI pipeline (including testing, linting, mypy typing, and pip-audit) has passed successfully for the develop branch.
2. **User Consent:** **NEVER** merge to the `master` branch without presenting the CI status to the USER and requesting explicit permission to merge. 
3. **Merge:** Only merge once explicit consent is received.
